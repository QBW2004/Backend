## 一、需求 ⑤ 在线用户监管增强

### 改动文件
- `TTY.Web/Areas/Game/Views/UserInfo/OnlinePlayers.cshtml`

### 落地内容
1. **新增聚合列**（复用已有字段，零数据库改动）：
   - 累计充值 = `COINS_BUY`（购币累计，`B_Users` 上分时 `COINS_BUY += coin`）
   - 累计兑换 = `COINS_BACK`（兑币累计，下分时 `COINS_BACK += coin`）
   - 累计盈亏 = `Profit`（控制器已计算 `COINS_BUY - COINS_BACK`），用 `fmtProfit` 着色：盈>0 红、亏<0 绿。

### 数据来源说明
- `GetOnlinePlayersList`（`B_Users.cs`）已 `select` `COINS_BUY/COINS_BACK/COINS/GAME_SCORE`，并经 `FillCurrentPlayInfo` 用最新 `UserOptLog` 填充当前游戏/厅/桌/座/下注/输赢；权限过滤沿用 `GetManagedAgencyAccounts`。本轮仅在视图层暴露既有数据，未改 BLL/控制器，风险最低。

## 二、需求 ⑥ 代理客服链接回退

### 改动文件
- `YYT.BLL/EF/B_Admin.cs`
- `TTY.Web/Controllers/Api/CustomerServiceUrlController.cs`

### 落地内容
1. **`B_Admin.ResolveCustomerServiceUrl(string agencyId)`**：解析玩家所属代理的客服链接，规则——
   - 先沿代理层级 `Admin.AGENCY`（指向上级账号）逐级**向上查找**，命中首个非空 `CustomerServiceUrl` 即返回；带 `visited` 去环 + 深度 64 防护。
   - 整条线为空 → **回退一**：平台默认（最高级管理员 `PRIV==0` 且已配置链接）。
   - 仍为空 → **回退二**：任意已配置链接（按 `PRIV` 由高到低）。
   - 全空返回空串。
2. **`CustomerServiceUrlController.Post`**：改为调用 `ResolveCustomerServiceUrl(users.AGENCY)`，并移除原先 `GetSingle(admin).CustomerServiceUrl` 的空引用风险（原逻辑当代理无记录或链接为空时会 NPE / 返回空）。

### 行为变化
- 旧：仅取玩家直属代理的客服链接，为空则前端拿到空。
- 新：直属为空时自动向上取上级 / 平台默认，保证前台「联系客服」始终可用。
